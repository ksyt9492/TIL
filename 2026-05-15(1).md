## apt update i386 경고 해결

### 증상
```
N: Skipping acquire of configured file 'main/binary-i386/Packages' as repository
'https://dl.google.com/linux/chrome-stable/deb stable InRelease' doesn't support architecture 'i386'
```

### 원인
- `/etc/apt/sources.list.d/google-chrome.sources` 파일에 `Architectures` 항목 없음
- apt가 i386 아키텍처도 탐색하려다 지원 안 된다는 경고 발생

### 해결
```bash
sudo sed -i '/^Components: main/a Architectures: amd64' /etc/apt/sources.list.d/google-chrome.sources
```

### 주의
- 파일 상단에 `### THIS FILE IS AUTOMATICALLY CONFIGURED ###` 주석 있음
- Chrome 업데이트 시 자동으로 덮어쓰여 경고가 재발할 수 있음
- 시스템 전체 i386 비활성화는 Wine 사용 가능성 때문에 보류

## DEB822 형식 (.sources)

- 기존 형식: `/etc/apt/sources.list.d/*.list`
- 신형 형식: `/etc/apt/sources.list.d/*.sources` (DEB822)
- 주요 필드: `Types`, `URIs`, `Suites`, `Components`, `Architectures`, `Signed-By`
